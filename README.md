# Event-Driven Architecture with API Gateway, EventBridge & Lambda

## Overview

Designed a scalable and loosely coupled event-driven architecture on AWS using API Gateway, EventBridge and Lambda to enable asynchronous processing and flexible system integration.

## Objective

* Build a loosely coupled event-driven system
* Enable asynchronous processing using serverless services
* Improve scalability and system extensibility
* Reduce dependency between services

## Problem Statement

* Tightly coupled systems are difficult to scale and maintain
* Synchronous communication leads to performance bottlenecks
* Limited flexibility in extending system functionality

## Architecture

* Amazon API Gateway receives incoming client requests
* Amazon EventBridge routes events based on defined rules
* AWS Lambda functions process events independently
* Fully serverless architecture ensures scalability and minimal operational overhead

## AWS Services Used

* Amazon API Gateway
* AWS Lambda
* Amazon EventBridge

## Outcome / Business Impact

* Reduced operational overhead with serverless architecture
* Improved scalability through event-driven design
* Enabled faster feature development and system extensibility
* Eliminated tight coupling between services
