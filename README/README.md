PROJECT 2

"EVENT-DRIVEN ARCHITECTURE WITH API GATEWAY, EVENTBRIDGE & LAMBDA"


OBJECTIVE:

Build a loosely coupled event-driven architecture

Enable asynchronous processing using AWS serverless services

Improve scalability and extensibility


PROBLEM STATEMENT:

Tightly coupled systems are hard to scale and maintain

Synchronous processing causes performance bottlenecks


ARCHITECTURE OVERVIEW:

API Gateway receives client requests

Event Bridge routes events based on rules

Lambda functions process events independently

Fully serverless architecture


AWS SERVICES USED:

Amazon API Gateway

AWS Lambda

Amazon Event Bridge


BUSINESS IMPACT:

Reduced operational overhead

Faster feature expansion

High scalability with no server management
